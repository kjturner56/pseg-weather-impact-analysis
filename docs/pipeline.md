# Pipeline Design

- **Load EIA Data:** Reads `pjm_daily_2025.csv` from the `data/` folder and loads it into a Pandas DataFrame for processing.
- **Load NOAA Data:** Reads `newark_weather_2025.csv` from the `data/` folder and loads it into a Pandas DataFrame for processing.
- **Clean EIA Data:** Converts the Date field to a standard datetime format, converts numeric fields to the appropriate data types, validates Demand, Forecast, Generation, and Interchange values, and outputs a cleaned EIA DataFrame.
- **Clean NOAA Data:** Converts the Date field to a standard datetime format, converts weather measurements to numeric values, calculates Average Temperature (TAVG) from TMAX and TMIN, checks for missing values, and outputs a cleaned NOAA DataFrame.
- **Merge Datasets:** Joins the cleaned EIA and NOAA datasets on the Date field to produce `merged_data.csv`.
- **Generate Metrics:** Calculates analytical metrics from the merged dataset to support the research question. Metrics include daily forecast error, forecast error percentage, monthly average electricity demand, monthly average temperature, Pearson correlations between electricity demand and TAVG, TMAX, and TMIN, comparative statistics by temperature category, and a normalized metric based on Temperature Stress. Outputs summary tables as CSV files for downstream visualization and reporting.
- **Create Visualizations:** Reads the merged dataset and summary metrics to generate labeled charts and figures. Visualizations are saved to the `output/` folder for inclusion in the final report.
- **Final Orchestration:** `final_project.py` executes the complete pipeline, integrates all components, and produces the final datasets, metrics, summary tables, and visualizations.
- **Interactive Dashboard:** `streamlit_app.py` loads the pipeline outputs and presents the results through an interactive Streamlit web application.

## Normalized Metric

The project calculates a normalized metric based on **Temperature Stress**, defined as the absolute difference between the daily average temperature (TAVG) and **65°F**.

**Temperature Stress**

```
Temperature Stress = abs(TAVG - 65)
```

The normalized metric is:

```
Demand per Degree of Temperature Stress = Demand / Temperature Stress
```

This metric normalizes electricity demand by the severity of daily weather conditions, allowing electricity demand to be compared across both hot and cold periods using a common baseline. The implementation will include appropriate handling for days where Temperature Stress equals zero.

## Temperature Categories

To support comparative analysis, daily average temperature (TAVG) will be classified into the following categories:

| Category | TAVG Range |
|----------|------------|
| Very Cold | < 32°F |
| Cold | 32°F – 49°F |
| Mild | 50°F – 64°F |
| Warm | 65°F – 79°F |
| Hot | ≥ 80°F |

These categories were selected because they better reflect changes in heating and cooling demand that influence electricity consumption. They provide a practical framework for comparing electricity demand and forecast accuracy under different weather conditions while supporting the project's operational focus.

The metrics component will calculate comparative statistics for each temperature category, including:

- Number of days
- Average electricity demand
- Average forecast error

These comparisons will support the operational recommendations presented in the final report.
